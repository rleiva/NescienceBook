\chapter{Inaccuracy}
\label{chap:Inaccuracy}

\begin{quote}
\begin{flushright}
\emph{A little inaccuracy sometimes saves\\
tons of explanations.}\\
Saki
\end{flushright}
\end{quote}
\bigskip


In Chapter \ref{chap:Mismodel}, we introduced mismodel as a theoretical measure of the discrepancy between a description currently available in scientific practice and an ideal description of the representation under study. In particular, the deficiency component of mismodel measures how much algorithmic information is still needed to transform a given description into a minimal description of the intended representation. From a formal point of view, this is the most natural way to quantify how incomplete or defective a description is.

However, this quantity cannot usually be approximated in practice. The main obstacle is that its definition refers to minimal descriptions of the representation \(r\), and such descriptions are generally unknown. In ordinary scientific practice, we do not have access to the set \(\mathcal D_r^\star\) of minimal descriptions of \(r\), nor do we know which existing model, if any, is closest to that set. There is also a deeper theoretical obstruction: minimal descriptions are determined by Kolmogorov complexity, and Kolmogorov complexity is non-computable. We therefore need computable, or at least approximable, proxies.

In this book we use two complementary approximations of mismodel. The first approximation, studied in this chapter, compares the output produced by the current description with the representation that the description is intended to model. If \(d\) is a description and \(\delta(d)\) is the string produced by executing it, then inaccuracy measures how far \(\delta(d)\) is from the target representation \(r\). In this sense, inaccuracy does not ask whether \(d\) is close to a minimal description of \(r\). It asks whether the output of \(d\) is close to \(r\). It is therefore an output-level approximation to the deficiency of a description.

The second approximation, studied in Chapter \ref{chap:Redundancy}, goes in a different direction. Instead of comparing the output of the description with the target representation, surfeit compares the description itself with the expected size of a minimal description. Thus, inaccuracy approximates the failure of a description to reproduce the intended representation, whereas surfeit approximates the failure of the description to be minimal as a description.

These two approximations capture different aspects of mismodel. A description may have low inaccuracy because it reproduces the representation correctly, while still having high surfeit because it is unnecessarily long or overfitted. Conversely, a description may be compact and have low surfeit, while still being inaccurate because its output fails to reproduce the intended representation. This distinction is essential for the theory of nescience: a good scientific model should be both accurate and economical.

\section{Inaccuracy}
\label{sec:inaccuracy:inaccuracy}

Let \(r \in \mathcal B^\ast\) be a representation and let \(d \in \mathcal D\) be a description. Since \(d\) is a valid description in the formal sense, executing it on the fixed universal Turing machine produces some finite string. We denote this output by
\[
r'=\delta(d).
\]
If \(r'=r\), then \(d\) is a valid description of \(r\). If \(r'\neq r\), then \(d\) is inaccurate with respect to \(r\).

The concept of inaccurate description has already been introduced in Chapter \ref{chap:Descriptions}. Here we use it to define a quantitative measure. The main question is the following: if the current description produces \(r'\) instead of \(r\), how much algorithmic information is needed to correct this discrepancy?

A first answer would be to measure \(K(r\mid r')\), the amount of information required to reconstruct the intended representation \(r\) from the output \(r'\) produced by the description. This captures missing information: the part of \(r\) that is not present in \(r'\). However, as in the case of miscoding, this one-sided measure is not sufficient. The output \(r'\) may also contain additional structure not present in \(r\). In that case, it is also relevant to consider \(K(r'\mid r)\), the amount of information needed to reconstruct the erroneous output from the intended representation.

For this reason, inaccuracy is defined using a bidirectional information distance between \(r\) and \(\delta(d)\).

\begin{definition}[Inaccuracy]
\label{def:inaccuracy:inaccuracy:inaccuracy}
Let \(r \in \mathcal B^\ast\) be a representation, and let \(d \in \mathcal D\) be a description. We define the \emph{inaccuracy}\index{Inaccuracy} of \(d\) with respect to \(r\), denoted by \(\iota(d,r)\), as
\[
\iota(d,r)
=
\frac{
\max\left\{
K\bigl(r\mid \delta(d)\bigr),
K\bigl(\delta(d)\mid r\bigr)
\right\}
}{
\max\left\{
K(r),
K\bigl(\delta(d)\bigr)
\right\}
}.
\]
\end{definition}

Thus, inaccuracy is the normalized information distance between the intended representation \(r\) and the representation \(\delta(d)\) produced by the description. The numerator measures the bidirectional correction cost between the two strings. The denominator normalizes this cost by the size of the larger of the two strings, so that inaccuracies associated with different representations can be compared on a common scale.

As usual in algorithmic information theory, equalities and inequalities involving Kolmogorov complexity should be understood up to additive constants independent of the particular strings under consideration. These constants depend on the choice of universal Turing machine and on the fixed encoding conventions.

The definition above is theoretically natural, but it is not directly computable. In practice, we approximate it by using the identity
\[
\max\{K(x\mid y),K(y\mid x)\}
=
K(x,y)-\min\{K(x),K(y)\}
+ O(\log \max\{K(x),K(y)\}),
\]
which holds for finite strings \(x,y\). Ignoring the logarithmic term for sufficiently large strings, the inaccuracy of \(d\) with respect to \(r\) can be approximated by
\[
\iota(d,r)
\approx
\frac{
K\bigl(r,\delta(d)\bigr)
-
\min\left\{
K(r),
K\bigl(\delta(d)\bigr)
\right\}
}{
\max\left\{
K(r),
K\bigl(\delta(d)\bigr)
\right\}
}.
\]
Replacing \(K\) by a computable estimate \(\widehat K\), we obtain the practical approximation
\[
\widehat{\iota}(d,r)
=
\frac{
\widehat K\bigl(r,\delta(d)\bigr)
-
\min\left\{
\widehat K(r),
\widehat K\bigl(\delta(d)\bigr)
\right\}
}{
\max\left\{
\widehat K(r),
\widehat K\bigl(\delta(d)\bigr)
\right\}
}.
\]
This is the form used in applications. Depending on the domain, \(\widehat K\) may be estimated by compression algorithms, empirical coding schemes, residual coding costs, or model-specific description lengths.

\begin{example}
Inaccuracy concerns the output of a description, not the internal form of the description itself.

Suppose that a dataset is generated by a quadratic function, but we describe it using a linear model. The description \(d\) corresponding to the linear model produces a predicted dataset \(\delta(d)\). The inaccuracy of \(d\) with respect to the observed representation \(r\) measures how difficult it is to transform the predicted dataset into the observed dataset, and conversely.

It does not measure how difficult it is to transform the linear model itself into the quadratic model. That second question belongs to the analysis of descriptions as descriptions, and is closer to the role played by surfeit in Chapter \ref{chap:Redundancy}.
\end{example}

The normalized definition makes inaccuracy comparable across representations of different sizes.

\begin{proposition}
\label{prop:inaccuracy:inaccuracy:relative_range}
For every representation \(r \in \mathcal B^\ast\) and every description \(d \in \mathcal D\),
\[
0 \leq \iota(d,r) \leq 1 + o(1),
\]
where the \(o(1)\) term vanishes as
\[
\max\{K(r),K(\delta(d))\}
\]
tends to infinity.
\end{proposition}

\begin{proof}
The numerator is non-negative because Kolmogorov complexities are non-negative. Therefore \(\iota(d,r)\geq 0\).

For the upper bound, let \(x=r\) and \(y=\delta(d)\). We use the standard inequalities
\[
K(x\mid y) \leq K(x)+O(1),
\qquad
K(y\mid x) \leq K(y)+O(1).
\]
Hence
\[
\max\{K(x\mid y),K(y\mid x)\}
\leq
\max\{K(x),K(y)\}+O(1).
\]
Dividing by \(\max\{K(x),K(y)\}\), we obtain
\[
\iota(d,r)
\leq
1+
\frac{O(1)}{\max\{K(x),K(y)\}}.
\]
For sufficiently complex strings, the additive term becomes negligible. Therefore \(0\leq \iota(d,r)\leq 1+o(1)\).
\end{proof}

In the rest of the book, and especially in applications, we treat inaccuracy as a normalized quantity in the interval \([0,1]\), ignoring machine-dependent additive constants.

Inaccuracy vanishes when the description produces the intended representation.

\begin{proposition}
\label{prop:inaccuracy_perfect_description}
Let \(r \in \mathcal B^\ast\) be a representation and \(d \in \mathcal D\) a description. If \(d \in \mathcal D_r\), then
\[
\iota(d,r)=O\left(\frac{1}{K(r)}\right).
\]
In particular, inaccuracy is zero up to the usual additive constants of Kolmogorov complexity.
\end{proposition}

\begin{proof}
If \(d \in \mathcal D_r\), then \(\delta(d)=r\). Therefore
\[
K(r\mid \delta(d))=K(r\mid r)=O(1),
\]
and
\[
K(\delta(d)\mid r)=K(r\mid r)=O(1).
\]
Hence the numerator of \(\iota(d,r)\) is \(O(1)\), while the denominator is \(K(r)\). Therefore
\[
\iota(d,r)=O\left(\frac{1}{K(r)}\right).
\]
\end{proof}

Conversely, if \(\iota(d,r)\) is close to zero, then \(r\) and \(\delta(d)\) are algorithmically close. This does not imply literal equality of the two strings, but it means that each can be reconstructed from the other using only a small amount of additional information.

\begin{proposition}
\label{prop:small_inaccuracy_algorithmic_closeness}
Let \(r \in \mathcal B^\ast\) and \(d \in \mathcal D\). If
\[
\iota(d,r) \leq \varepsilon,
\]
then
\[
K(r\mid \delta(d))
\leq
\varepsilon
\max\{K(r),K(\delta(d))\}
\]
and
\[
K(\delta(d)\mid r)
\leq
\varepsilon
\max\{K(r),K(\delta(d))\}.
\]
\end{proposition}

\begin{proof}
The result follows immediately from the definition of \(\iota(d,r)\), since the numerator is the maximum of the two conditional complexities.
\end{proof}

This proposition clarifies the meaning of small inaccuracy. A description with low inaccuracy produces an output that is close to the target representation in both directions: the output contains almost all the algorithmic information needed to reconstruct \(r\), and it contains little independent information not already present in \(r\).

Finally, inaccuracy extends immediately to joint representations. If \(r,s \in \mathcal B^\ast\), their joint representation is the concatenated or self-delimited string \(\langle r,s\rangle\). We then define
\[
\iota(d,\langle r,s\rangle)
=
\frac{
\max\left\{
K\bigl(\langle r,s\rangle\mid \delta(d)\bigr),
K\bigl(\delta(d)\mid \langle r,s\rangle\bigr)
\right\}
}{
\max\left\{
K(\langle r,s\rangle),
K\bigl(\delta(d)\bigr)
\right\}
}.
\]
No new concept is required: the joint representation is itself a finite string.

\section{Inaccuracy as an Approximation to Description Deficiency}
\label{sec:inaccuracy_deficiency_approximation}

The main role of inaccuracy in the theory of nescience is to approximate the deficiency of a description. Recall that the deficiency of a description measures how much information must be added to a given description in order to obtain a minimal description of the target representation. If \(\mathcal D_r^\star\) denotes the set of minimal descriptions of \(r\), then the deficiency of \(d\) with respect to \(r\) is of the form
\[
\operatorname{def}_r(d)
=
\min_{d^\star \in \mathcal D_r^\star}
K(d^\star \mid d).
\]
This is a description-level quantity: it compares \(d\) with a minimal program for \(r\).

By contrast, inaccuracy is an output-level quantity: it compares \(\delta(d)\) with \(r\). It does not ask whether the program \(d\) is close to a minimal program. It asks whether the output produced by \(d\) is close to the intended representation. Therefore, inaccuracy is not identical to description deficiency. It is a computable proxy for one of its observable consequences.

The relation between the two quantities can be explained as follows. If the output \(\delta(d)\) is far from \(r\), then any description obtained by correcting \(d\) must somehow provide the missing information needed to recover \(r\). Thus, high inaccuracy is evidence of description deficiency. If \(\delta(d)\) is close to \(r\), then only a small amount of output-level correction is needed, and the description is accurate even if it may still be unnecessarily long.

The following proposition gives a simple upper bound on the cost of producing some valid description of \(r\) from \(d\).

\begin{proposition}
\label{prop:inaccuracy_valid_description_bound}
Let \(r \in \mathcal B^\ast\) and \(d \in \mathcal D\). There exists a valid description \(\tilde d \in \mathcal D_r\) such that
\[
K(\tilde d \mid d)
\leq
K(r\mid \delta(d)) + O(1).
\]
\end{proposition}

\begin{proof}
Given \(d\), we can compute \(\delta(d)\). Let \(p\) be a shortest program that transforms \(\delta(d)\) into \(r\). Thus
\[
l(p)=K(r\mid \delta(d)).
\]
Construct a new description \(\tilde d\) that first executes \(d\), obtains \(\delta(d)\), then applies \(p\) to obtain \(r\), and finally outputs \(r\). This construction is computable from \(d\) and \(p\), up to a fixed overhead independent of \(r\) and \(d\). Therefore
\[
K(\tilde d\mid d)
\leq
l(p)+O(1)
=
K(r\mid \delta(d))+O(1).
\]
Since \(\tilde d\) outputs \(r\), we have \(\tilde d\in\mathcal D_r\).
\end{proof}

This proposition shows that the one-sided correction term \(K(r\mid \delta(d))\) bounds the amount of information needed to transform the current description into some valid description of \(r\). However, it does not guarantee that the resulting description is minimal. The correction may produce a valid but highly redundant description.

This is precisely why inaccuracy cannot replace mismodel. It only measures whether the output is correct or nearly correct. It does not measure whether the description achieves the minimal form required for zero mismodel.

\begin{proposition}
\label{prop:zero_inaccuracy_not_zero_mismodel}
A description may have zero inaccuracy with respect to a representation \(r\) while still having positive surfeit, and hence positive mismodel.
\end{proposition}

\begin{proof}
Let \(d\in\mathcal D_r\) be any valid description of \(r\) that is not minimal. Then \(\delta(d)=r\), so the inaccuracy of \(d\) with respect to \(r\) is zero up to the usual additive constants. However, because \(d\) is not minimal, there exists a shorter description \(d^\star \in \mathcal D_r^\star\). Hence \(d\) contains unnecessary descriptive content relative to a minimal description, and its surfeit is positive. Therefore its mismodel need not be zero.
\end{proof}

This proposition is important for interpreting inaccuracy correctly. A highly accurate model may still be a poor scientific model if it achieves its accuracy through excessive complexity. This is the familiar problem of overfitting in machine learning, but the point is more general: correctness of output is not the same as economy of explanation.

The next proposition gives the complementary observation. Low surfeit does not imply low inaccuracy.

\begin{proposition}
\label{prop:low_surfeit_not_low_inaccuracy}
A description may have low surfeit while still having high inaccuracy.
\end{proposition}

\begin{proof}
Let \(d\) be a very short description that outputs a simple string \(s\), and let \(r\) be a complex representation unrelated to \(s\). The description \(d\) may be close to minimal for its own output \(s\), and hence have low surfeit as a description of \(s\). Nevertheless, if \(s=\delta(d)\) is algorithmically far from \(r\), then
\[
K(r\mid s)
\]
will be large, and the inaccuracy \(\iota(d,r)\) will be high. Therefore low descriptive economy does not guarantee accuracy with respect to the intended representation.
\end{proof}

Together, these propositions justify the separation between inaccuracy and surfeit. Inaccuracy estimates whether the current description produces the right representation. Surfeit estimates whether the description is unnecessarily complex. Both are required to approximate mismodel in practice.

\section{Practical Approximation of Inaccuracy}
\label{sec:practical_approximation_inaccuracy}

The definition of inaccuracy relies on Kolmogorov complexity and is therefore non-computable. Nevertheless, it is more accessible in practice than mismodel deficiency, because it does not require identifying minimal descriptions. Instead, it only requires comparing two strings: the target representation \(r\) and the output \(\delta(d)\) produced by the description.

In applications, we approximate Kolmogorov complexity by a computable function \(\widehat K\). This approximation depends on the domain. For textual representations, \(\widehat K\) may be the length of a compressed file. For empirical datasets, \(\widehat K\) may be the length of an empirical code. For predictive models, \(\widehat K(r\mid \delta(d))\) may be approximated by the amount of information needed to encode the residuals between the observed data and the model predictions.

The normalized approximation used throughout this book is
\[
\widehat{\iota}(d,r)
=
\frac{
\widehat K\bigl(r,\delta(d)\bigr)
-
\min\left\{
\widehat K(r),
\widehat K\bigl(\delta(d)\bigr)
\right\}
}{
\max\left\{
\widehat K(r),
\widehat K\bigl(\delta(d)\bigr)
\right\}
}.
\]
This expression is useful because it avoids the direct estimation of conditional Kolmogorov complexities. Instead, it estimates the shared and non-shared information between the two strings using their individual and joint compressed lengths.

\begin{definition}[Estimated Inaccuracy]
\label{def:estimated_inaccuracy}
Let \(r \in \mathcal B^\ast\) be a representation, \(d\in\mathcal D\) a description, and \(\widehat K\) a computable approximation of Kolmogorov complexity. We define the \emph{estimated inaccuracy}\index{Estimated inaccuracy} of \(d\) with respect to \(r\) as
\[
\widehat{\iota}(d,r)
=
\frac{
\widehat K\bigl(r,\delta(d)\bigr)
-
\min\left\{
\widehat K(r),
\widehat K\bigl(\delta(d)\bigr)
\right\}
}{
\max\left\{
\widehat K(r),
\widehat K\bigl(\delta(d)\bigr)
\right\}
}.
\]
\end{definition}

The quality of \(\widehat{\iota}\) depends on the quality of \(\widehat K\). If the compressor or coding method captures the regularities relevant to the representation, then the estimate can be meaningful. If the compressor fails to capture those regularities, then the estimate may be misleading.

\begin{proposition}
\label{prop:estimated_inaccuracy_consistency}
Suppose that \(\widehat K\) approximates \(K\) with error at most \(\varepsilon\) on the strings \(r\), \(\delta(d)\), and \(\langle r,\delta(d)\rangle\). Then the estimated inaccuracy \(\widehat{\iota}(d,r)\) differs from the ideal normalized information-distance form by a quantity bounded by a term depending on \(\varepsilon\) and on
\[
\max\{K(r),K(\delta(d))\}.
\]
In particular, for large representations and small relative compression error, \(\widehat{\iota}(d,r)\) is a stable approximation.
\end{proposition}

\begin{proof}
The estimated numerator differs from the ideal numerator only through the approximation errors in the joint complexity and in the two individual complexities. Since the operations involved are addition, subtraction, minimum, and maximum, the accumulated error is bounded by a constant multiple of \(\varepsilon\). The denominator is similarly perturbed by at most \(\varepsilon\). Dividing by a large denominator makes the relative effect of these errors small. Therefore the estimate becomes stable when the strings are sufficiently large and the approximation errors are small relative to their complexities.
\end{proof}

The practical interpretation is straightforward. Estimated inaccuracy is reliable only relative to a chosen encoding scheme. This is not a defect, but a necessary feature of the theory: in applications, every measurement of algorithmic information must be mediated by a concrete coding procedure.

\begin{example}
In a regression problem, the representation \(r\) may be a dataset of observed response values, while \(\delta(d)\) is the vector of predictions produced by a model. The inaccuracy of the model can be estimated by encoding the residuals between observations and predictions. If the residuals are small and structured, the correction cost is low. If the residuals are large or irregular, the correction cost is high.

This generalizes the ordinary notion of prediction error. Instead of measuring only numerical deviation, inaccuracy measures the amount of information needed to correct the model output into the observed representation.
\end{example}

\section{Conditional Inaccuracy}
\label{sec:conditional_inaccuracy}

In this section, we delve deeper into the concept of inaccuracy by considering its application to conditional descriptions. Specifically, we explore the inaccuracy of a description when evaluated in conjunction with pre-existing background knowledge, a notion we term \emph{conditional inaccuracy}. As we will see, the inaccuracy of a conditional description can never exceed that of its unconditional counterpart; at worst, it remains unchanged. This property makes conditional inaccuracy a useful tool for evaluating new concepts or models and for assessing their explanatory power with respect to the entity of interest.

In Definition \ref{def:conditional_description}, we introduced the concept of a conditional description \(d\) for a representation \(r\), given an arbitrary background string \(s\). This is denoted by \(d \mid s\) and defined as the self-delimiting concatenated string \(\langle d, s \rangle\), where \(d = \langle TM, a \rangle\) and \(TM(\langle a, s \rangle) = r\). If \(TM(\langle a, s \rangle) = r'\) with \(r \neq r'\), then \(d \mid s\) is referred to as an \emph{inaccurate} conditional description of \(r\).

It is important to note that \(d \mid s\) must be defined for all possible background strings \(s\). Additionally, we refer to the case in which \(s\) is the empty string, denoted \(d \mid \lambda\), as the \emph{unconditional} version of the description.

Building on the concept of inaccuracy introduced in Definition \ref{def:inaccuracy:inaccuracy:inaccuracy}, we now define the notion of \emph{conditional inaccuracy} to capture the error introduced when using an inaccurate conditional description.

\begin{definition}
Let \(r \in \mathcal{B}^\ast\) be a representation, \(s \in \mathcal{B}^\ast\) a background string, and \(d \mid s\) an inaccurate conditional description. We define the \emph{conditional inaccuracy}\index{Conditional inaccuracy} of the description \(d\) for the representation \(r\) given the string \(s\), denoted by \(\iota(d \mid s, r)\), as:
\[
\iota(d \mid s, r) = \frac{ \max\left\{ K \left(r \mid \delta(d \mid s) \right),\ K \left( \delta(d \mid s) \mid r \right) \right\} }{ \max\left\{ K(r),\ K \left( \delta(d \mid s) \right) \right\} }.
\]
\end{definition}

Conditional inaccuracy is thus defined as the normalized information distance between the original representation \(r\) and the string produced by the conditional description \(d \mid s\).

As a normalized measure, the conditional inaccuracy of a description lies within the interval \([0, 1]\), up to the usual additive constants of Kolmogorov complexity.

\begin{proposition}
\label{prop:range_conditional_inaccuracy}
Let \(r \in \mathcal{B}^\ast\) be a representation, \(s \in \mathcal{B}^\ast\) a string, and \(d \mid s\) a conditional description of \(r\) given \(s\). Then
\[
0 \leq \iota(d \mid s, r) \leq 1+o(1).
\]
\end{proposition}

\begin{proof}
This follows directly from the same argument used in Proposition \ref{prop:inaccuracy:inaccuracy:relative_range}, applied to the strings \(r\) and \(\delta(d\mid s)\).
\end{proof}

The conditional inaccuracy assumes a value of zero if the conditional description \(d \mid s\) correctly produces the representation \(r\).

\begin{proposition}
\label{prop:perfect_conditional_description}
Let \(r \in \mathcal{B}^\ast\) be a representation, \(s \in \mathcal{B}^\ast\) a string, and \(d \mid s\) a conditional description of \(r\) given \(s\), where \(d = \langle TM, a \rangle\). If
\[
TM \left(\langle a, s \rangle \right) = r,
\]
then
\[
\iota(d \mid s, r)=O\left(\frac{1}{K(r)}\right).
\]
\end{proposition}

\begin{proof}
If \(TM \left(\langle a, s \rangle \right) = r\), then \(\delta(d \mid s) = r\), which implies that
\[
K \left( r \mid \delta(d \mid s) \right) = O(1)
\]
and
\[
K \left( \delta(d \mid s) \mid r \right) = O(1).
\]
The numerator is therefore \(O(1)\), while the denominator is \(K(r)\). Hence
\[
\iota(d \mid s,r)=O\left(\frac{1}{K(r)}\right).
\]
\end{proof}

Incorporating established prior knowledge into research should not increase the inaccuracy of a description when that prior knowledge is used appropriately. If the background knowledge is relevant to the representation being described, the description may use it to produce a more accurate output. If the background knowledge is irrelevant, it can be ignored.

\begin{theorem}
\label{th:conditional_inaccuracy}
Let \(r \in \mathcal{B}^\ast\) be a representation, and let \(d \in \mathcal{D}\) be a conditional description of \(r\). Then, under the assumption that the conditional description can ignore irrelevant background strings,
\[
\iota(d \mid s, r) \leq \iota(d , r)
\]
for all strings \(s \in \mathcal{B}^\ast\), up to the usual additive constants.
\end{theorem}

\begin{proof}
Since \(\iota(d,r)\) is equivalent to \(\iota(d\mid \lambda,r)\), it is enough to compare the output produced with background string \(s\) and the output produced with the empty background string \(\lambda\).

If the conditional description can ignore irrelevant background information, then the computation using \(s\) can simulate the computation using \(\lambda\). Moreover, conditioning on additional information cannot increase Kolmogorov complexity beyond an additive constant:
\[
K(r\mid \langle \delta(d),s\rangle)
\leq
K(r\mid \delta(d))+O(1).
\]
Thus, when \(s\) is useful, it may reduce the correction cost; when it is not useful, it can be ignored. Therefore the conditional inaccuracy cannot exceed the unconditional inaccuracy, up to the usual constants.
\end{proof}

Theorem \ref{th:conditional_inaccuracy} represents a foundational intuition in the theory of nescience. It supports the idea that background knowledge can reduce the cost of producing accurate descriptions. In practical applications, one typically focuses on prior knowledge directly related to the entity under study. However, the theorem also suggests that exploratory use of background knowledge from other domains need not be harmful, provided the description can ignore irrelevant information.

\begin{example}
The P versus NP problem\index{P vs. NP} is one of the most important open problems in theoretical computer science. Constructing a complete solution from scratch would require an enormous amount of information. However, a conditional description may rely on background knowledge from complexity theory, formal languages, computability theory, and proof theory. Such background knowledge does not solve the problem by itself, but it can reduce the amount of information that must be explicitly included in any proposed description of a solution.
\end{example}

Finally, conditional inaccuracy extends immediately to joint representations. Given two representations \(r,t \in \mathcal B^\ast\), we may write
\[
\iota(d \mid s,\langle r,t\rangle)
=
\frac{
\max\left\{
K \left(\langle r,t\rangle \mid \delta(d \mid s) \right),
K \left( \delta(d \mid s) \mid \langle r,t\rangle \right)
\right\}
}{
\max\left\{
K(\langle r,t\rangle),
K \left(\delta(d \mid s) \right)
\right\}
}.
\]
Again, no new concept is needed, since \(\langle r,t\rangle\) is itself a finite string.

\section{Decreasing Inaccuracy}
\label{sec:decreasing_inaccuracy}

The purpose of defining inaccuracy is not merely to measure the error of a description, but to guide its reduction. Given a target representation \(r\), we may compare two candidate descriptions \(d_1,d_2\in\mathcal D\) by examining whether the second produces an output closer to \(r\) than the first.

\begin{definition}
Let \(r \in \mathcal B^\ast\) be a representation, and let \(d_1,d_2\in\mathcal D\) be two descriptions. We define the \emph{absolute variation of inaccuracy}\index{Variation of inaccuracy} from \(d_1\) to \(d_2\), with respect to \(r\), as
\[
\Delta_{\iota}^{a}(d_1,d_2,r)
=
\iota(d_1,r)-\iota(d_2,r).
\]
\end{definition}

If \(\Delta_{\iota}^{a}(d_1,d_2,r)>0\), then \(d_2\) is more accurate than \(d_1\) with respect to \(r\). If \(\Delta_{\iota}^{a}(d_1,d_2,r)<0\), then the new description is less accurate. Since inaccuracy is normalized, the absolute variation lies between \(-1\) and \(1\), up to the usual constants.

It is often useful to express the improvement relative to the initial inaccuracy.

\begin{definition}
Let \(r \in \mathcal B^\ast\) be a representation, and let \(d_1,d_2\in\mathcal D\) be two descriptions with \(\iota(d_1,r)\neq 0\). We define the \emph{relative variation of inaccuracy}\index{Relative variation of inaccuracy} from \(d_1\) to \(d_2\), with respect to \(r\), as
\[
\Delta_{\iota}^{r}(d_1,d_2,r)
=
\frac{\iota(d_1,r)-\iota(d_2,r)}
{\iota(d_1,r)}.
\]
\end{definition}

A value of \(0\) indicates no change. A value of \(1\) indicates complete elimination of inaccuracy. Negative values indicate that the new description is less accurate than the previous one.

Relative variation must be interpreted carefully. When \(\iota(d_1,r)\) is already very small, a tiny absolute change can produce a large relative variation. Thus, both absolute and relative improvements should be considered.

A second way to reduce inaccuracy is to change the representation rather than the description. This may be useful when the current representation is difficult to model. However, changing the representation may also affect miscoding, since the new representation may encode the entity better or worse than the original one.

\begin{definition}
Let \(r_1,r_2 \in \mathcal B^\ast\) be two representations, and let \(d \in \mathcal D\) be a description. We define the \emph{absolute variation of inaccuracy} from \(r_1\) to \(r_2\), with respect to \(d\), as
\[
\Delta_{\iota}^{a}(d,r_1,r_2)
=
\iota(d,r_1)-\iota(d,r_2).
\]
\end{definition}

A positive value means that the fixed description \(d\) is more accurate with respect to \(r_2\) than with respect to \(r_1\). This comparison does not say that \(r_2\) is a better representation of the entity. It only says that \(d\) models \(r_2\) more accurately. Whether this change is scientifically desirable also depends on the miscoding of the representations.

\begin{definition}
Let \(r_1,r_2 \in \mathcal B^\ast\) be two representations, and let \(d \in \mathcal D\) be a description with \(\iota(d,r_1)\neq 0\). We define the \emph{relative variation of inaccuracy} from \(r_1\) to \(r_2\), with respect to \(d\), as
\[
\Delta_{\iota}^{r}(d,r_1,r_2)
=
\frac{\iota(d,r_1)-\iota(d,r_2)}
{\iota(d,r_1)}.
\]
\end{definition}

This quantity measures the proportional reduction in inaccuracy obtained by replacing \(r_1\) with \(r_2\) while keeping the description fixed.

Inaccuracy can always be reduced trivially by choosing the representation to be the output of the description. If \(r_2=\delta(d)\), then \(\iota(d,r_2)=0\). But such a move may dramatically increase miscoding, because \(r_2\) may be a poor representation of the entity. This observation illustrates why inaccuracy cannot be optimized in isolation.

\begin{proposition}
\label{prop:trivial_inaccuracy_reduction}
For every description \(d\in\mathcal D\), there exists a representation \(r\in\mathcal B^\ast\) such that
\[
\iota(d,r)=0
\]
up to additive constants.
\end{proposition}

\begin{proof}
Take \(r=\delta(d)\). Then \(d\) outputs \(r\), so \(d\in\mathcal D_r\). By Proposition \ref{prop:inaccuracy_perfect_description}, the inaccuracy is zero up to additive constants.
\end{proof}

This proposition shows why the theory of nescience must consider miscoding, inaccuracy, and surfeit together. A model can be made perfectly accurate by redefining the target representation to match its output, but that does not mean that the resulting hypothesis represents the intended entity correctly.

\section{Inaccuracy-Miscoding Rate of Change}
\label{sec:Inaccuracy_Miscoding_Rate_of_Change}

In the previous section we considered two ways to reduce inaccuracy: changing the description while keeping the representation fixed, and changing the representation while keeping the description fixed. In scientific practice, both components may change simultaneously. A new hypothesis may introduce a new representation and a new description.

Let
\[
\mathbf{x}_1=(r_1,d_1),
\qquad
\mathbf{x}_2=(r_2,d_2)
\]
be two hypotheses about the same entity. Moving from \(\mathbf{x}_1\) to \(\mathbf{x}_2\) may reduce inaccuracy while increasing miscoding, or reduce miscoding while increasing inaccuracy. This produces a trade-off.

\begin{definition}
Let \(e \in \mathcal E\) be an entity, and let
\[
\mathbf{x}_1=(r_1,d_1), \qquad \mathbf{x}_2=(r_2,d_2)
\]
be two hypotheses. We define the \emph{inaccuracy-miscoding rate of change}\index{Inaccuracy-miscoding rate of change} from \(\mathbf{x}_1\) to \(\mathbf{x}_2\) as
\[
\Delta_{\iota\mu}(\mathbf{x}_1,\mathbf{x}_2)
=
\frac{
\iota(d_2,r_2)-\iota(d_1,r_1)
}{
\mu(r_2)-\mu(r_1)
},
\]
provided that \(\mu(r_2)-\mu(r_1)\neq 0\).
\end{definition}

The ratio measures how much inaccuracy changes per unit change in miscoding. If the denominator is small, the ratio can become unstable, so it should always be interpreted together with the absolute changes
\[
\iota(d_2,r_2)-\iota(d_1,r_1)
\]
and
\[
\mu(r_2)-\mu(r_1).
\]

\begin{example}
Suppose that
\[
\iota(d_1,r_1)=0.40,
\qquad
\mu(r_1)=0.15,
\]
and
\[
\iota(d_2,r_2)=0.20,
\qquad
\mu(r_2)=0.25.
\]
Then inaccuracy decreases by \(0.20\), while miscoding increases by \(0.10\). The rate of change is
\[
\Delta_{\iota\mu}(\mathbf{x}_1,\mathbf{x}_2)
=
\frac{0.20-0.40}{0.25-0.15}
=
\frac{-0.20}{0.10}
=
-2.
\]
The negative value indicates a trade-off: the new hypothesis is more accurate, but its representation is more miscoded.
\end{example}

\begin{example}
Suppose instead that
\[
\iota(d_1,r_1)=0.35,
\qquad
\mu(r_1)=0.20,
\]
and
\[
\iota(d_2,r_2)=0.30,
\qquad
\mu(r_2)=0.2001.
\]
Then
\[
\Delta_{\iota\mu}(\mathbf{x}_1,\mathbf{x}_2)
=
\frac{0.30-0.35}{0.2001-0.20}
=
\frac{-0.05}{0.0001}
=
-500.
\]
The ratio is large in magnitude, but the actual changes are small. This illustrates why rate-of-change quantities should not be used without also inspecting the underlying absolute variations.
\end{example}

The inaccuracy-miscoding trade-off is a two-objective projection of the full nescience problem. In the full theory, we must also consider surfeit. A hypothesis that reduces both miscoding and inaccuracy may still be undesirable if it introduces a large amount of unnecessary descriptive complexity.

\section{Inaccuracy of Areas}
\label{sec:inaccuracy_areas}

The notion of inaccuracy can be extended from individual representations to research areas. Let \(\mathcal A\subseteq \mathcal E\) be a research area, and suppose that a finite subset of effectively available representations is given by
\[
\widehat{\mathcal A}
=
\{r_1,r_2,\ldots,r_n\}.
\]
We encode these representations into a single self-delimited string
\[
\langle r_1,r_2,\ldots,r_n\rangle.
\]
A description of the area is then evaluated according to how accurately it reproduces this joint representation.

\begin{definition}
Let \(\mathcal A \subseteq \mathcal E\) be a research area with known finite representation set
\[
\widehat{\mathcal A}
=
\{r_1,r_2,\ldots,r_n\},
\]
and let \(d\in\mathcal D\) be a description. We define the \emph{inaccuracy of the area}\index{Inaccuracy of an area} given \(d\), denoted by \(\iota(d,\widehat{\mathcal A})\), as
\[
\iota(d,\widehat{\mathcal A})
=
\frac{
\max\left\{
K\left(\langle r_1,r_2,\ldots,r_n\rangle \mid \delta(d)\right),
K\left(\delta(d)\mid \langle r_1,r_2,\ldots,r_n\rangle\right)
\right\}
}{
\max\left\{
K(\langle r_1,r_2,\ldots,r_n\rangle),
K(\delta(d))
\right\}
}.
\]
\end{definition}

This definition treats the available representations of the area as a single composite representation. The description \(d\) is accurate for the area if its output is algorithmically close to that composite string.

\begin{proposition}
\label{prop:inaccuracy:inaccuracy:range}
For every finite known subset
\[
\widehat{\mathcal A}
=
\{r_1,r_2,\ldots,r_n\}
\]
and every description \(d\in\mathcal D\),
\[
0\leq \iota(d,\widehat{\mathcal A})\leq 1+o(1).
\]
\end{proposition}

\begin{proof}
The string \(\langle r_1,r_2,\ldots,r_n\rangle\) is an element of \(\mathcal B^\ast\). The result therefore follows by applying Proposition \ref{prop:inaccuracy:inaccuracy:relative_range} to the two strings
\[
\langle r_1,r_2,\ldots,r_n\rangle
\]
and
\[
\delta(d).
\]
\end{proof}

The inaccuracy of an area measures how much information is needed to correct a description of a collection of related representations. In practical applications, this may be useful when a scientific field is represented by a corpus of documents, a family of datasets, a group of models, or a structured collection of experimental results.

\section*{References}

A good introduction to the study of uncertainties and error analysis in science, particularly in physics, chemistry, and engineering, is the textbook by Taylor \cite{taylor2022introduction}. Another accessible reference on measurement and uncertainty, aimed at undergraduate students in science and technology, is the book by Hughes and Hase \cite{hughes2010measurements}.

The formal background for the definition of inaccuracy comes from algorithmic information theory and, in particular, from the theory of Kolmogorov complexity and normalized information distance. The standard reference is the book by Li and Vitányi \cite{li2008introduction}. These ideas also connect naturally with the minimum description length principle and related coding-based approaches to statistical learning.

From a philosophical perspective, Popper's work \cite{popper2014conjectures} remains relevant because it emphasizes the role of error, refutation, and correction in scientific progress. In the present framework, inaccuracy provides an algorithmic way to quantify one specific kind of error: the discrepancy between what a description produces and the representation it is intended to reproduce.






In Section \ref{sec:descriptions_models}, we introduced the notion of a description, or model, of an entity as a computer program. When executed, this program reproduces one of the representations encoding the entity in question. More precisely, a description $d$ for a representation $r$ of an entity $e$ is a Turing machine that produces the string $r$ when interpreted by a universal Turing machine $\delta$. However, due to our typically incomplete understanding of the entity $e$, the actual output of the description, denoted as $r' = \delta(d)$, will generally resemble but not exactly match $r$.

In this chapter, we investigate the error introduced by flawed models, specifically, how closely the output $r'$ approximates the intended representation $r$. We refer to this type of error as the \emph{inaccuracy} of the description $d$.

Inaccuracy constitutes the second metric for assessing our understanding of a research entity. The underlying idea is that the more accurate our model, the closer $r'$ is to $r$, and thus the better our understanding of the entity. Formally, the inaccuracy of a description $d$ is defined as the normalized information distance between the original representation $r$ and the output representation $r'$ produced by the description. That is, inaccuracy measures the length of the shortest computer program needed to transform the erroneous output $r'$ into the correct representation $r$.

Inaccuracy evaluates how well the output of a description aligns with the selected representation encoding the entity. However, as discussed in the preceding chapter, the representation itself may be flawed. Inaccuracy focuses exclusively on the description $d$, without accounting for potential miscoding in the representation $r$. Moreover, although its computation does not require an oracle, inaccuracy cannot always be calculated exactly in practice; instead, it must often be estimated—a topic we will address in Part II of this book.

In this chapter, we formally define inaccuracy and examine its key properties. We also analyze how inaccuracy behaves when conditional descriptions are used in place of unconditional ones. Finally, we extend the notion of inaccuracy from individual entities to entire research areas.

This investigation is not purely theoretical, it has significant practical relevance. Accurate models are essential across a wide range of domains, from climate prediction to the development of artificial intelligence systems. Understanding and quantifying inaccuracy can thus lead to better models, ultimately improving our ability to make reliable predictions and informed decisions.

\section{Inaccuracy}
\label{sec:inaccuracy:inaccuracy}

In the process of studying an entity $e \in \mathcal{E}$ through a representation $r \in \mathcal{R}_e$, we may encounter situations in which our proposed description $d$ fails to accurately produce $r$. That is, $d \notin \mathcal{D}_r$ (see Definition \ref{def:descriptions_model}). In such cases, when the universal Turing machine $\delta$ receives $d$ as input, it produces a string $r'$ that differs from the original representation $r$.

Intuitively, one might say that $d$ is an inaccurate description of the entity $e$. However, because descriptions refer to entities only indirectly, via representations, our formal notion of inaccuracy must be defined in terms of the representation, not the entity itself. Furthermore, we must account for the possibility that the representation $r$ is itself flawed, as previously discussed through the concept of miscoding.

With these considerations in mind, we introduce the following definition of an inaccurate description.

\begin{definition}
Let $r \in \mathcal{B}^\ast$ be a representation, and let $d \in \mathcal{D}$ be a description, where $d = \langle TM, a \rangle$. If the output of $TM(a)$ is a string $r'$ such that $r \neq r'$, we say that $d$ is an \emph{inaccurate}\index{Inaccurate description} description for $r$.
\end{definition}

Our proposed description $d$ may fall outside the set of valid descriptions $\mathcal{D}_r$ for $r$ (indicating positive inaccuracy), and the representation $r$ may not belong to the set of valid representations $\mathcal{R}^\star_e$ for the entity $e$ (indicating positive miscoding).

When a description is inaccurate, we aim to quantify the degree of inaccuracy. Within the computational framework, a natural approach is to measure how difficult it is to transform the incorrect representation $r'$, obtained by executing $d$ on the universal Turing machine, into the original representation $r$. This difficulty is captured by the normalized information distance between $r'$ and $r$.

\begin{definition} [Inaccuracy]
\label{def:inaccuracy:inaccuracy:inaccuracy}
Let $r \in \mathcal{B}^\ast$ be a representation, and let $d \in \mathcal{D}$ be a description, where $d = \langle TM, a \rangle$. We define the \emph{inaccuracy}\index{Inaccuracy} of the description $d$ with respect to the representation $r$, denoted by $\iota(d, r)$, as:
\[
\iota(d, r) = \frac{ \max\left\{ K \left(r \mid \delta(d) \right),\; K \left( \delta(d) \mid r \right) \right\} }{ \max\left\{ K(r),\; K \left(\delta(d) \right) \right\} }
\]
\end{definition}

The use of a relative measure of inaccuracy, rather than an absolute one, enables meaningful comparisons between inaccuracies across different descriptions of the same representation, as well as between descriptions of different representations.

Similar to miscoding (see Definition \ref{def:miscoding}), inaccuracy is computed using a bidirectional approach: we calculate the length of the shortest computer program that can generate the correct representation $r$ from the erroneous one $r'$, and vice versa, that is, the shortest program that can generate $r'$ from $r$. In essence, a valid description should produce a representation that contains all the necessary information to reconstruct the intended entity, while excluding any erroneous or irrelevant content.

\begin{example}
Inaccuracy primarily concerns the difficulty of correcting the \emph{output} of a description, that is, the result produced by a computable model, rather than the difficulty of modifying the description itself.

For example, suppose we have a dataset generated by a system that is perfectly described by a quadratic function, but we choose a linear function as our model. In this case, inaccuracy evaluates how different the predicted dataset (produced by the linear model) is from the original quadratic dataset. It does not measure how difficult it is to transform the incorrect linear model into the correct quadratic one.

In this sense, if the original dataset consists of 10 points, a polynomial of degree ten that perfectly fits the data would also yield an inaccuracy of zero. Determining which model is preferable, the quadratic model with zero inaccuracy or the more complex degree-ten polynomial, also with zero inaccuracy, is a matter addressed by the surfeit metric (see Chapter \ref{chap:Redundancy}).
\end{example}

Given its basis in Kolmogorov complexity, inaccuracy is a quantity that, in general, cannot be computed exactly and must instead be approximated. The method used to approximate inaccuracy depends on the specific characteristics of the entities under study and the nature of their representations.

Conveniently, the inaccuracy of a description always falls within the range $[0, 1]$, as established by the following proposition.

\begin{proposition}
\label{prop:inaccuracy:inaccuracy:relative_range}
For all representations $r \in \mathcal{B}^\ast$ and all descriptions $d \in \mathcal{D}$, it holds that $0 \leq \iota(d, r) \leq 1$.
\end{proposition}
\begin{proof}
This follows from the general inequality
\[
0 \leq \frac{ \max\{ K(x \mid y),\ K(y \mid x) \} }{ \max\{ K(x),\ K(y) \} } \leq 1
\]
for all $x, y \in \mathcal{B}^\ast$, as stated in Proposition \ref{prop:ncd_between_zero_and_one}.
\end{proof}

The proposition above applies to all pairs of descriptions $d$ and representations $r$, even in cases where $d$ is not intended to model $r$. In such instances, the inaccuracy $\iota(d, r)$ will typically be close to one.

Inaccuracy is exactly zero if and only if the description $d$ is one of the valid descriptions of the representation $r$.

\begin{proposition}
\label{prop:inaccuracy_perfect_description}
Given a description $d \in \mathcal{D}$ and a representation $r \in \mathcal{B}^\ast$, we have that $\iota(d, r) = 0$ if and only if $d \in \mathcal{D}_r$, i.e., $d$ is a valid description of $r$.
\end{proposition}
\begin{proof}
If $d \in \mathcal{D}_r$, then by definition, $\delta(d) = r$. Consequently, we have $K \left( r \mid \delta(d) \right) = K \left( \delta(d) \mid r \right) = 0$, and thus $\iota(d, r) = 0$.

Conversely, suppose $\iota(d, r) = 0$. Then,
\[
\max\left\{ K \left( r \mid \delta(d) \right),\ K \left( \delta(d) \mid r \right) \right\} = 0,
\]
which implies that both conditional complexities are zero. Therefore, $r = \delta(d)$, and it follows that $d \in \mathcal{D}_r$.
\end{proof}

Finally, given two representations $r$ and $s$, we may also be interested in evaluating the inaccuracy of a description $d$ when it is used to describe their joint representation $rs$. Since we require that $rs$ is itself a valid representation, the extension of the inaccuracy concept to joint representations is straightforward and does not require a new definition:
\[
\iota(d, rs) = \frac{ \max\left\{ K \left(rs \mid \delta(d) \right),\ K \left( \delta(d) \mid rs \right) \right\} }{ \max\left\{ K(rs),\ K \left(\delta(d) \right) \right\} }
\]
As a direct consequence of Proposition \ref{prop:inaccuracy:inaccuracy:range}, for any representations $r, s \in \mathcal{B}^\ast$ and any description $d \in \mathcal{D}$, we have:
\[
0 \leq \iota(d, rs) \leq 1.
\]

\section{Conditional Inaccuracy}

In this section, we delve deeper into the concept of inaccuracy by considering its application to conditional descriptions. Specifically, we explore the inaccuracy of a description when evaluated in conjunction with pre-existing background knowledge, a notion we term \emph{conditional inaccuracy}. As we will see, the inaccuracy of a conditional description can never exceed that of its unconditional counterpart; at worst, it remains unchanged. This property makes conditional inaccuracy a useful tool for evaluating new concepts or models and for assessing their explanatory power with respect to the entity of interest.

In Definition \ref{def:conditional_description}, we introduced the concept of a conditional description $d$ for a representation $r$, given an arbitrary background string $s$. This is denoted by $d \mid s$ and defined as the self-delimiting concatenated string $\langle d, s \rangle$, where $d = \langle TM, a \rangle$ and $TM(\langle a, s \rangle) = r$. If $TM(\langle a, s \rangle) = r'$ with $r \neq r'$, then $d \mid s$ is referred to as an \emph{inaccurate} conditional description of $r$.

It is important to note that $d \mid s$ must be defined for all possible background strings $s$. Additionally, we refer to the case in which $s$ is the empty string, denoted $d \mid \lambda$, as the \emph{unconditional} version of the description.

Building on the concept of inaccuracy introduced in Definition \ref{def:inaccuracy:inaccuracy:inaccuracy}, we now define the notion of \emph{conditional inaccuracy} to capture the error introduced when using an inaccurate conditional description.

\begin{definition}
Let $r \in \mathcal{B}^\ast$ be a representation, $s \in \mathcal{B}^\ast$ a background string, and $d \mid s$ an inaccurate conditional description. We define the \emph{conditional inaccuracy}\index{Conditional inaccuracy} of the description $d$ for the representation $r$ given the string $s$, denoted by $\iota(d \mid s, r)$, as:
\[
\iota(d \mid s, r) = \frac{ \max\left\{ K \left(r \mid \delta(d \mid s) \right),\ K \left( \delta(d \mid s) \mid r \right) \right\} }{ \max\left\{ K(r),\ K \left( \delta(d \mid s) \right) \right\} }
\]
\end{definition}

Conditional inaccuracy is thus defined as the normalized information distance between the original representation $r$ and the string produced by the conditional description $d \mid s$.

As a normalized measure, the conditional inaccuracy of a description lies within the interval $[0, 1]$.

\begin{proposition}
\label{prop:range_conditional_inaccuracy}
Let $r \in \mathcal{B}^\ast$ be a representation, $s \in \mathcal{B}^\ast$ a string, and $d \mid s$ a conditional description of $r$ given $s$. Then $0 \leq \iota(d \mid s, r) \leq 1$.
\end{proposition}
\begin{proof}
This follows directly from the general inequality:
\[
0 \leq \frac{ \max\{ K(x \mid y),\ K(y \mid x) \} }{ \max\{ K(x),\ K(y) \} } \leq 1
\]
for all $x, y \in \mathcal{B}^\ast$, as established in Proposition \ref{prop:ncd_between_zero_and_one}.
\end{proof}

The conditional inaccuracy assumes a value of zero if and only if the conditional description $d \mid s$ is a valid model that correctly produces the representation $r$.

\begin{proposition}
\label{prop:perfect_conditional_description}
Let $r \in \mathcal{B}^\ast$ be a representation, $s \in \mathcal{B}^\ast$ a string, and $d \mid s$ a conditional description of $r$ given $s$, where $d = \langle TM, a \rangle$. Then $\iota(d \mid s, r) = 0$ if and only if $TM \left(\langle a, s \rangle \right) = r$.
\end{proposition}
\begin{proof}
If $TM \left(\langle a, s \rangle \right) = r$, then $\delta(d \mid s) = r$, which implies that
\[
K \left( r \mid \delta(d \mid s) \right) = K \left( \delta(d \mid s) \mid r \right) = 0.
\]
Hence, $\iota(d \mid s, r) = 0$. Conversely, if $\iota(d \mid s, r) = 0$, then
\[
\max\left\{ K \left( r \mid \delta(d \mid s) \right),\ K \left( \delta(d \mid s) \mid r \right) \right\} = 0,
\]
which implies both conditional complexities are zero. Therefore, $\delta(d \mid s) = r$, which means $TM \left(\langle a, s \rangle \right) = r$.
\end{proof}

Incorporating established prior knowledge into research does not increase the inaccuracy of a description. If this background knowledge is relevant to the representation being described, the oracle will use it accordingly. Conversely, if the prior knowledge is irrelevant, the oracle will simply disregard it. The following theorem formalizes this idea.

\begin{theorem}
\label{th:conditional_inaccuracy}
Let $r \in \mathcal{B}^\ast$ be a representation, and $d \in \mathcal{D}$ a conditional description of $r$. Then
\[
\iota(d \mid s, r) \leq \iota(d , r)
\]
for all strings $s \in \mathcal{B}^\ast$.
\end{theorem}
\begin{proof}
Since $\iota(d, r)$ is equivalent to $\iota(d \mid \lambda, r)$, we need to prove that
\[
\frac{ \max\{ K \left(r \mid \delta(d \mid s) \right), K \left( \delta(d \mid s) \mid r \right) \} } { \max\{ K(r), K \left( \delta(d \mid s) \right) \} } \leq \frac{ \max\{ K \left(r \mid \delta(d \mid \lambda ) \right), K \left( \delta(d \mid \lambda ) \mid r \right) \} } { \max\{ K(r), K \left( \delta(d \mid \lambda ) \right) \} }.
\]
This inequality follows from the fact that
\[
K \left(r \mid \langle \delta(d), s \rangle \right) \leq K \left(r \mid \delta(d) \right),
\]
as shown in Proposition \ref{prop:kolmogorov_conditional}.
\end{proof}

Theorem \ref{th:conditional_inaccuracy} represents a foundational result in the theory of nescience. It establishes the basis for developing a robust methodology aimed at deepening our understanding (i.e., reducing inaccuracy) of a research entity. In practical applications, our primary focus will typically be on prior knowledge directly related to the subject of study. However, the core insight of Theorem \ref{th:conditional_inaccuracy} is that incorporating concepts from seemingly unrelated domains will not compromise the accuracy of our investigation. This theorem becomes particularly powerful when such exploratory processes are automated (see Chapter \ref{chap:computational-creativity}).

\begin{example}
The P vs. NP problem\index{P vs. NP} stands as one of the most significant unresolved questions in computer science. It asks whether every problem whose solution can be verified in polynomial time (class NP) can also be solved in polynomial time (class P). The relationship between these two complexity classes remains unsolved. Constructing a comprehensive, self-contained solution to this problem in a formal language is an immense challenge. However, leveraging relevant prior knowledge can significantly reduce the complexity of the required description. For instance, insights from Algorithm Theory, which deals with the classification and efficiency of algorithms, and from Formal Language Theory, which addresses the structure of computational problems (e.g., regular and context-free languages) and highlights the role of Turing machines, can be instrumental. Drawing upon such established knowledge may not only simplify our descriptions but also facilitate a deeper understanding and potentially contribute to resolving the P vs. NP problem.
\end{example}

Finally, given two representations $r$ and $t$, the formalization of the concept of conditional inaccuracy, when applied to the joint representation $rt$, is straightforward and does not require a new definition:
\[
\iota(d \mid s) = \frac{ \max\{ K \left(rt \mid \delta(d \mid s) \right), K \left( \delta(d \mid s) \mid rt \right) \} } { \max\{ K(rt), K \left(\delta(d \mid s) \right) \} }
\]
As a normalized measure, $\iota(d \mid s)$ always takes a value in the interval $[0, 1]$.

\section{Decreasing Inaccuracy}

Our objective is to reduce the inaccuracy of the current description $d_1$, thereby improving our understanding of the original entity. This improvement may involve either modifying $d_1$ to correct or eliminate its inaccuracies, or developing a completely new description based on a different approach to modeling the entity. In either case, the result is a new description $d_2$. In this section, we aim to analyze how the introduction of a new description $d_2$ affects the inaccuracy compared to the original description $d_1$.

\begin{definition}
Let $r \in \mathcal{B}^\ast$ be a representation, and let $d_1, d_2 \in \mathcal{D}$ be two descriptions. We define the \emph{variation of inaccuracy}\index{Variation of inaccuracy} between the descriptions $d_1$ and $d_2$, with respect to $r$, denoted by $\Delta^{a}_{\iota} ( d_1, d_2, r )$, as:
\[
\Delta^{a}_{\iota} ( d_1, d_2, r ) = \iota(d_1, r) - \iota(d_2, r)
\]
\end{definition}

Since inaccuracy is bounded between 0 and 1, the maximum possible variation in inaccuracy is $\pm 1$. A positive value of $\Delta_{\iota}$ indicates that description $d_2$ is preferable to $d_1$ in terms of accuracy. Conversely, a negative value suggests that $d_1$ is more accurate than $d_2$. It is important to note that the new description may also introduce a substantial increase in surfeit, potentially outweighing the improvement in inaccuracy. For a detailed discussion of surfeit, refer to Chapter \ref{chap:Redundancy}, and for an explanation of how inaccuracy and surfeit combine into the unified metric of nescience, see Chapter \ref{chap:Nescience}.

We can also introduce a relative measure of the variation in inaccuracy, when moving from description $d_1$ to $d_2$

\begin{definition}
Let $r \in \mathcal{B}^\ast$ be a representation, and $d_1, d_2 \in \mathcal{D}$ be two descriptions. We define the \emph{relative variation of inaccuracy}\index{Relative variation of inaccuracy} between descriptions $d_1$ and $d_2$, denoted by $\Delta^{r}_{\iota}(d_1, d_2, r)$, as:
\[
\Delta^{r}_{\iota}(d_1, d_2, r) = \frac{\iota(d_1, r) - \iota(d_2, r)}{\iota(d_1, r)}
\]
provided that $\iota(d_1, r) \neq 0$.
\end{definition}

A value of $0$ indicates no change, while a value of $1$ corresponds to a complete elimination of inaccuracy. Negative values, on the other hand, signify an increase in inaccuracy, indicating that the new description $d_2$ is less accurate than the original $d_1$. Note that the relative variation can be arbitrarily negative, diverging to $-\infty$ as $\iota(d_2, r)$ increases and $\iota(d_1, r)$ approaches zero.

As inaccuracy approaches zero, relative variations become increasingly unstable. A small absolute change can produce a large relative variation if the initial inaccuracy is very low. For instance, if $\iota(d_1, r) = 0.1$ and the inaccuracy decreases by 0.05, this corresponds to a relative improvement of 50\%. In contrast, if $\iota(d_1, r) = 0.9$ and the same absolute improvement occurs, the relative reduction is only about 5.6\%. Both absolute and relative variations are essential for evaluating the significance and magnitude of improvements.

An alternative strategy for reducing uncertainty about an entity involves modifying its representation rather than altering its description. While such a change might increase the miscoding of the entity, the potential reduction in inaccuracy could outweigh this drawback.

\begin{definition}
Let $r_1, r_2 \in \mathcal{B}^\ast$ be two representations, and let $d \in \mathcal{D}$ be a description. We define the \emph{variation of inaccuracy}\index{Variation of inaccuracy} between representations $r_1$ and $r_2$, denoted by $\Delta^{a}_{\iota}(d, r_1, r_2)$, as:
\[
\Delta^{a}_{\iota}(d, r_1, r_2) = \iota(d, r_1) - \iota(d, r_2)
\]
\end{definition}

As before, since inaccuracy is bounded between 0 and 1, the maximum possible variation is 1. A positive value of $\Delta^{a}_{\iota}$ indicates a preference for representation $r_2$ over $r_1$ with respect to the fixed description $d$. Conversely, a negative value suggests that $r_1$ is more accurate than $r_2$. This comparison is based solely on inaccuracy, as miscoding is not considered here. Additionally, since the description remains unchanged, there is no risk of variation in surfeit.

Finally, we can also introduce a relative variation of inaccuracy with respect to a change in representation

\begin{definition}
Let $r_1, r_2 \in \mathcal{B}^\ast$ be two representations, and $d \in \mathcal{D}$ be a description. We define the \emph{relative variation of inaccuracy} of the representations $r_1, r_2$, denoted by $\Delta^{r}_{\iota} ( d, r_1, r_2 )$, as:
\[
\Delta^{r}_{\iota} ( d, r_1, r_2 ) = \frac{\iota(d, r_1) - \iota(d, r_2)}{\iota(d, r_1)}
\]
provided that $\iota(d, r_1) \neq 0$.
\end{definition}

This quantity measures the proportional reduction in inaccuracy resulting from replacing representation $r_1$ with $r_2$, while keeping the description fixed. A value of $0$ indicates no change in inaccuracy, and a value of $1$ corresponds to a complete elimination of inaccuracy. However, $\Delta^{r}_{\iota}$ can also take negative values, potentially diverging to $-\infty$, when the new representation $r_2$ performs worse than $r_1$. As the inaccuracy of the initial representation $r_1$ approaches zero, even minor absolute changes in inaccuracy can lead to large swings in the relative variation, making it increasingly volatile.

\section{Inaccuracy-Miscoding Rate of Change}
\label{sec:Inaccuracy_Miscoding_Rate_of_Change}

In the preceding section, we examined how the inaccuracy of a representation can be reduced by selecting a different description. We also explored an alternative strategy in which inaccuracy is minimized not by modifying the description, but by changing the representation itself. In this section, we turn our attention to a more general approach for reducing the inaccuracy associated with an entity. Rather than altering the description or the representation in isolation, it may be more effective to modify both simultaneously.

The balance between the amount of miscoding we are willing to accept in order to achieve a reduction in inaccuracy is referred to as the \emph{miscoding-inaccuracy trade-off}. For a broader discussion of trade-offs in multi-objective optimization, refer to Section \ref{sec:trade_offs}.

\begin{definition}
Let $e \in \mathcal{E}$ be an entity, and $\mathbf{x}_1, \mathbf{x}_2 \in \mathcal{R}_e \times \mathcal{D}$ be two hypotheses, with $\mathbf{x}_1 = (r_1, d_1)$ and $\mathbf{x}_2 = (r_2, d_2)$. We define the \emph{rate of change between the inaccuracy and the miscoding} of the hypothesis $\mathbf{x}_1, \mathbf{x}_2$, denoted by $\Delta_{\mu \iota} ( \mathbf{x}_1, \mathbf{x}_2 )$ as:
\[
\Delta_{\iota \mu} ( \mathbf{x}_1, \mathbf{x}_2 ) = \frac{\iota(d_2, r_2) - \iota(d_1, r_1)}{\mu(r_2) - \mu(r_1)}
\] 
provided that $\mu(r_2) - \mu(r_1) \neq 0$.
\end{definition}

The ratio $\Delta_{\iota \mu}$ represents the rate of change between inaccuracy and miscoding when transitioning from the first hypothesis to the second. A positive value of $\Delta_{\iota \mu}$ implies that both quantities, miscoding and inaccuracy, either decrease (which is desirable) or increase (which is undesirable). The interpretation becomes more nuanced when $\Delta_{\iota \mu}$ is negative, indicating that one of the quantities decreases while the other increases. In such cases, two scenarios must be considered:

\begin{enumerate}[label=(\roman*)]
\item Inaccuracy decreases and miscoding increases, we aim for $\Delta_{\iota \mu}(\mathbf{x}_1, \mathbf{x}_2) < M$, where $M < -1$, thereby ensuring that the reduction in inaccuracy compensates for the increase in miscoding.
\item Inaccuracy increases and miscoding decreases, we aim for $\Delta_{\iota \mu}(\mathbf{x}_1, \mathbf{x}_2) > M$, where $-1 < M < 0$, thereby ensuring that the reduction in miscoding justifies the increase in inaccuracy.
\end{enumerate}

In both cases, caution is warranted when the change in miscoding is small, as it can disproportionately affect the ratio and potentially lead to misleading conclusions.

\begin{example}
Let $\mathbf{x}_1$ and $\mathbf{x}_2$ be two hypotheses. For $\mathbf{x}_1$, the inaccuracy $\iota(d_1, r_1)$ is 0.40, and the miscoding $\mu(r_1)$ is 0.15. For $\mathbf{x}_2$, the inaccuracy $\iota(d_2, r_2)$ is 0.20 (a decrease from 0.40), and the miscoding $\mu(r_2)$ is 0.25 (an increase from 0.15). Using the definition of the rate of change, we compute:
\[
\Delta_{\iota \mu} ( \mathbf{x}_1, \mathbf{x}_2 ) = \frac{0.40 - 0.20}{0.15 - 0.25} = \frac{0.20}{-0.10} = -2
\]
In this case, transitioning from hypothesis $\mathbf{x}_1$ to $\mathbf{x}_2$ results in a decrease of inaccuracy by 0.20 units and an increase in miscoding by 0.10 units. The rate of change is $-2$.
\end{example}

Having a very small change in miscoding can significantly amplify the value of the rate of change, potentially giving the misleading impression that inaccuracy and miscoding are varying at an extreme rate, even when the actual changes are minor. This phenomenon is illustrated in the following example.

\begin{example}
Let $\mathbf{x}_1$ be a hypothesis with an inaccuracy $\iota(d_1, r_1)$ of 0.35 and a miscoding $\mu(r_1)$ of 0.20. Let $\mathbf{x}_2$ be a second hypothesis with an inaccuracy $\iota(d_2, r_2)$ of 0.30 (a slight decrease from 0.35) and a miscoding $\mu(r_2)$ of 0.2001 (a very small increase from 0.20). Applying the definition of the rate of change:
\[
\Delta_{\iota \mu} ( \mathbf{x}_1, \mathbf{x}_2 ) = \frac{0.35 - 0.30}{0.20 - 0.2001} = \frac{0.05}{-0.0001} = -500
\]
The rate of change is $-500$, which may misleadingly suggest a dramatic shift. In reality, the inaccuracy only decreased by 0.05 units, and the miscoding increased by a negligible 0.0001 units. The extremely small denominator inflates the result, making the change appear far more significant than it actually is.
\end{example}

As we attempt to optimize both inaccuracy and miscoding, we inevitably encounter configurations (hypotheses) where improving one objective necessitates compromising the other. The set of such "best trade-off" configurations constitutes the Pareto frontier (see Section \ref{sec:multiobjective_optimization}). Points on the Pareto frontier are said to be Pareto optimal because any attempt to improve one objective leads to a deterioration in the other.

\begin{definition}
Let \( e \in \mathcal{E} \) be an entity, and let \( \mathbf{x} = (r, d) \in \mathcal{R}_e \times \mathcal{D} \) be a hypothesis. The hypothesis \( \mathbf{x} \) is said to be a Pareto point\index{Pareto point} with respect to inaccuracy \( \iota \) and miscoding \( \mu \) if there does not exist another hypothesis \( \mathbf{x'} = (r', d') \) such that:
\begin{enumerate}
    \item \( \iota(d', r') \leq \iota(d, r) \) and \( \mu(r') \leq \mu(r) \), and
    \item \( \iota(d', r') < \iota(d, r) \) or \( \mu(r') < \mu(r) \).
\end{enumerate}
\end{definition}

In simpler terms, a hypothesis $\mathbf{x}$ is Pareto optimal if: (i) no other hypothesis is better in both inaccuracy and miscoding, and (ii) any improvement in one metric necessarily results in a worsening of the other.

The rate of change $\Delta_{\iota \mu}$ between any two Pareto optimal points provides insight into how the trade-off between inaccuracy and miscoding evolves along the Pareto frontier. If decision-makers are more sensitive to changes in inaccuracy than to miscoding, they may prefer configurations with a less negative $\Delta_{\iota \mu}$. Conversely, if miscoding is of greater concern, they may accept solutions where $\Delta_{\iota \mu}$ indicates a larger increase in inaccuracy in exchange for a smaller gain in miscoding.

\section{Inaccuracy of Areas}
\label{sec:inaccuracy_areas}

An area $\mathcal{A}$ (see Section \ref{sec:areas}) is a subset $\mathcal{A} \subset \mathcal{E}$ of entities that are related or share a common property. The concept of inaccuracy can be extended to research areas in order to quantitatively measure the effort required to correct an inaccurate description of an area.

Given the strings $r_1, r_2, \ldots, r_n$, where each $r_i \in \mathcal{B}^\ast$ for $i = 1, 2, \ldots, n$, recall that we use $\langle r_1, r_2, \ldots, r_n \rangle$ to denote a self-delimited encoding of the individual strings $r_i$ into a unified string, such that the original components can be fully recovered.

The following definition extends the concept of inaccuracy to research areas.

\begin{definition}
Let $\mathcal{A} \subset \mathcal{E}$ be an area with a known subset $\hat{\mathcal{A}} = {r_1, r_2, \ldots, r_n}$, and let $d \in \mathcal{D}$ be a description. We define the \emph{inaccuracy of the area} given the description $d$, denoted by $\iota(d, \hat{\mathcal{A}})$, as:
\[
\iota(d, \hat{\mathcal{A}}) = \frac{ \max\{ K \left( \langle r_1, r_2, \ldots, r_n \rangle \mid \delta(d) \right), K \left( \delta(d) \mid \langle r_1, r_2, \ldots, r_n \rangle \right) \} } { \max\{ K(\langle r_1, r_2, \ldots, r_n \rangle), K \left(\delta(d) \right) \} }
\]
\end{definition}

The inaccuracy of a description for an area falls within the interval from $0$ to $1$, as established by the following proposition.

\begin{proposition}
\label{prop:inaccuracy:inaccuracy:range}
For all known subsets $\hat{\mathcal{A}} = {r_1, r_2, \ldots, r_n}$ and all descriptions $d \in \mathcal{D}$, we have that $0 \leq \iota(d, \hat{\mathcal{A}}) \leq 1$.
\end{proposition}
\begin{proof}
The result follows directly from the fact that $\langle r_1, r_2, \ldots, r_n \rangle$ is a string in $\mathcal{B}^\ast$, and from Proposition \ref{prop:ncd_between_zero_and_one}.
\end{proof}

By extending the concept of inaccuracy to cover areas, we can quantitatively evaluate the quality of a description for a specific subset of entities. This mathematical framework provides a rigorous tool for assessing and correcting inaccuracies not only at the level of individual entities but also across broader research domains.
